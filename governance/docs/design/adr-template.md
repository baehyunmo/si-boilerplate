> **ADR 작성 가이드**
> 1. ADR은 하나의 아키텍처 결정 사항만을 다룬다.
> 2. 컨텍스트에는 결정이 필요한 시점의 상황을 사실 기반으로 기술한다.
> 3. 대안은 최소 2개 이상 검토하고, 비교 근거를 명확히 한다.
> 4. 결정 후에도 상황이 변경되면 새로운 ADR을 작성하여 기존 ADR을 대체한다.
> 5. ADR ID는 순차적으로 부여한다 (ADR-0001, ADR-0002, ...).
> 6. 상태 변경 시 문서 상단의 상태값을 갱신하고 이력을 기록한다.
